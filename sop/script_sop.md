# 创作 SOP 知识库

## 核心创作哲学

### 用户思维
- **用户不存在，用户只是需求的集合**
- 不要一个视频想让所有人看
- 需求决定约束 → 约束决定内容方向 → 决定外化的利益点 → 转化动作
- 思考模型：我是谁，凭什么，帮什么人，用什么方式，怎么解决，什么问题，达到什么效果

### 价值交换公式
```
产品为用户提供价值 = (新效用 - 旧效用) - (直接成本 + 交易成本 + 间接成本)
```

**新效用**（要具体化、可视化）：
- 成绩提升的确定性路径
- 学习痛苦感的降低
- 时间和精力的节省
- 归属感和认同感

**旧效用**（引发共鸣）：
- 学校课堂的弊端
- 自学的弊端
- 现有解决方案的局限

**直接成本**：金钱、时间、精力（用"投资"而非"花费"）

**交易成本**（摩擦）：认知成本、获取成本、使用成本、售后成本

**间接成本**：机会成本（最重要！）、心理压力与焦虑

## 开头设计

### 开头的职能
让用户完成"第一次时间付费"，解决四个疑虑：
1. "这关我什么事？"——与我无关的顾虑
2. "这有用吗？"——价值不明的顾虑
3. "你谁啊？我凭什么信你？"——信任缺失的顾虑
4. "会不会很复杂/耗时？"——成本过高的顾虑

长视频 = 无数短视频的组合，每15秒给用户一个续费理由。

### 开头写法：去匹配而不是圈定
- **谁**：角色身份、环境、属性、目标
- **在什么阶段**：追逐目标的阶段、时间维度、自身成长
- **纠结什么事**：最怕失去什么、最想避免什么风险

## 循环框架（CREM）

表达框架是无数循环的组合：
```
观点(C) → 原理(R) → 案例(E) → 方法(M) → 下一个观点...
```

**观点(C)**：核心论断，打破认知
**原理(R)**：解释为什么，让批判有据可依
**案例(E)**：具体场景/数据，让读者看到自己
**方法(M)**：可操作的解决步骤

## 标题封面设计

**如果用AI生成，关键词是"对立"**

核心要素组合：
- **情绪钩子**：触发焦虑、好奇、渴望、共鸣
- **认知颠覆**：打破常规认知，反直觉
- **价值承诺**：明确能获得什么
- **身份认同**：让目标受众感觉"这说的就是我"
- **方法论暗示**：暗示有可操作的系统方法

## 金句打磨体系

公式 = 找矛盾 → 套模型 → 磨节奏

**对比模型**：A是大家的底线，B是你自己的战线
**重复模型**：越XX，就越YY
**比喻模型**：做A就像做B（反差要大）
**极端模型**：对XX最危险/残酷的XX就是XX
**定义模型**：所谓的XX，不过是YY

## 营销转化检验清单

- [ ] 是否把"难受"翻译成可感知的"成本"（时间、情感、健康）？
- [ ] 是否点明了痛苦的本质（金句/观点）？
- [ ] 是否用具体例子解释了方法如何起效？
- [ ] 是否有明确的效率提升对比（如1小时抵3小时）和成果数据？
- [ ] 是否将产品描述为"高价值、可感知的解决方案组合包"？
- [ ] 是否将"点击链接"包装成"低风险、高回报"的理性决策？
- [ ] 行动指令是否足够具体、清晰、有力？
- [ ] 结尾是否给了用户积极的心理暗示？

## 写作细节

### 语言原则
- **动词 > 名词 > 形容词**
- 任何形容词都会有更好的具体行为+感受来替代
- 真实发生过的情感是最佳素材
- 数据比总结更有力
- **白描永远是最疼的刀**

### 结构原则
- 尽量养成2句话、2句话写文案的习惯
- 优秀的文案是美观的，因为循环结构就是逻辑嵌套

### 自查方法
写完文案读3遍，用配音软件听1遍：
- 50%：检查基本流畅度
- 20%：从【功能】出发完善逻辑链
- 20%：从【简洁】出发删除繁琐
- 10%：从【价值】出发做取舍

| 罪 | 应用 |

## B站转化逻辑

仅有三次展示机会：
1. 封面+标题（用户点击前）
2. 视频稿件
3. 评论区置顶

每个曝光位不要重复信息，根据用户当前掌握的信息状态传递最需要的内容。
